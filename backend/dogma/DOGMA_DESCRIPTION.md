# DOGMA DESCRIPTION

## Obiettivo

Creare una descrizione estesa che chiarisca valore, contesto d'uso e affidabilita.

## Vincoli

- 3-5 paragrafi brevi (niente elenco bullet mascherato).
- Integrare USP, obiezioni e target quando disponibili.
- Evitare keyword stuffing e ripetizioni inutili.
- Restare entro min/max chars configurati.

## Sequenza suggerita

1. Apertura: bisogno cliente + proposta del prodotto.
2. Sviluppo: dettagli tecnici e benefici.
3. Gestione dubbi: compatibilita, uso, limiti dichiarati.
4. Chiusura: rassicurazione concreta, senza CTA aggressiva.
